## Documentation style

Documentation and code comments must be written in British English, while code and identifiers must be written in US English.

Karsk is a proper noun, and as such must be capitalised when referring to this project. The CLI command must be written in all lower-case.

## Python Code Style

We use [Google style](https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings) for Python docstrings.
